# 首次最小值下标扫描要同时保留“最小值”和“首次出现”两层前缀语义

对这类题：

- 只读整数数组
- 单循环从左到右扫描
- 状态量不是“当前最小值”，而是“当前最小值的下标”
- 返回值要求是全局最小值的**第一次出现位置**

循环 invariant 不能只写“`a[idx]` 不大于已处理前缀里的所有元素”。这只能说明 `idx` 指向某个前缀最小值，不能说明它是**最左边**那个最小值。

常见症状：

- `symexec` 提示循环路径缺断言，说明前缀语义根本不够；
- 或 `symexec` 能过，但 return witness / loop-step witness 只能证明“是某个最小值”，证不出“更早位置都严格更大”。

更稳的规则是把 invariant 写成两层：

1. `idx` 在已处理前缀 `[0, i)` 中取到最小值；
2. `idx` 之前的所有位置都**严格大于**这个最小值。

最小形状：

```c
/*@ Inv
      1 <= i &&
      i <= n &&
      0 <= idx &&
      idx < i &&
      a == a@pre &&
      n == n@pre &&
      (forall (j: Z), (0 <= j && j < i) => l[idx] <= l[j]) &&
      (forall (j: Z), (0 <= j && j < idx) => l[j] > l[idx]) &&
      IntArray::full(a, n, l)
*/
```

为什么要分这两层：

- 第一层负责“当前下标确实对应前缀最小值”；
- 第二层负责“如果前面有相等元素，就不能把当前 `idx` 叫作第一次出现”；
- 分支 `if (a[i] < a[idx]) idx = i;` 只在**严格变小**时更新，正好对应第二层的严格不等式。

proof 里的典型保持性拆分也会直接跟着这两层 invariant 走：

- 更新分支：对旧前缀任意 `j < i`，用 `l[idx_old] <= l[j]` 和 `l[i] < l[idx_old]` 推出 `l[i] < l[j]`；
- 不更新分支：对新增位置 `j = i`，直接用 `l[i] >= l[idx]`；
- 退出时由 `i == n` 把前缀语义提升成整数组语义。

适用范围：

- `array_index_of_min`
- `array_index_of_max` 的“第一次出现”版本（把不等号方向整体翻转）
- 任何“返回前缀/全局极值的首次位置”的只读单扫描数组题
