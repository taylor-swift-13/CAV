## Issue 2: Coq proof tactic mismatches
- **Gate**: coqc proof_manual.v
- **Location**: `coq/generated/p023_strlen_proof_manual.v`
- **Symptom**: Multiple failures:
  1. `entailer!` solved some trivial safety/entail goals completely, leaving "No such goal" when followed by additional tactics.
  2. `split_pure_spatial` failed on deeply nested `&&` right-hand sides (e.g., `entail_wit_2`).
  3. `rewrite app_Znth1/app_Znth2` followed by `replace (n - n) with 0` failed because after `app_Znth2` the expression was `n - Zlength l`, not `n - n`.
  4. `dump_pre_spatial` failed on `return_wit_1` due to nested `&&` structure.
- **Fix**:
  - For trivial goals, use `pre_process.` alone (or `pre_process; entailer!` when subgoals remain).
  - For nested `&&` goals, use `entailer!` instead of `split_pure_spatial`.
  - For `app_Znth2` rewrites, use `replace (n - Zlength l) with 0` instead of `replace (n - n) with 0`.

## Issue 3: Duplicate lemma names in proof_auto.v and proof_manual.v
- **Gate**: coqc goal_check.v
- **Location**: `coq/generated/p023_strlen_goal_check.v:6`
- **Symptom**: `Error: The label proof_of_p023_strlen_safety_wit_1 is already declared.`
- **Root cause**: `proof_manual.v` defined all 8 lemmas, but `proof_auto.v` (generated by symexec) already contained Admitted stubs for 6 of them. Including both modules in `goal_check.v` caused name collisions.
- **Fix**: Removed from `proof_manual.v` all lemmas that were already present in `proof_auto.v`, keeping only `entail_wit_2` and `return_wit_1` which required manual proof.
