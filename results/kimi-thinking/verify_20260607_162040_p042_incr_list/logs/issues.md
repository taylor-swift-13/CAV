# Issues Encountered and Resolutions

## 1. No `Znth_map` lemma in library
**Problem:** To prove the pointwise property in `problem_42_spec`, we needed `Znth j (map_incr input_l) 0 = Znth j input_l 0 + 1`. There was no existing `Znth_map` lemma in the library.

**Resolution:** Proved two auxiliary lemmas manually:
- `Zlength_map_incr`: `Zlength (map_incr l) = Zlength l`
- `Znth_map_incr`: `forall j l, 0 <= j < Zlength l -> Znth j (map_incr l) 0 = Znth j l 0 + 1`

The `Znth_map_incr` proof used induction on `l`, case analysis on `j = 0`, and the `Znth_cons` lemma.

## 2. Rewrite in spatial context fails for general equalities
**Problem:** Rewriting `Zlength output_prefix = i` inside the separation logic context (`**`) fails because there is no `Proper` instance for `Zlength` under `sepcon`.

**Resolution:** Used `subst i` after proving `i = l_size_pre` by `lia`, which eliminated the variable entirely and avoided the rewrite in spatial context.

## 3. `split_pure_spatial` creates two subgoals requiring bullet discipline
**Problem:** After solving the spatial part with `entailer!`, the pure subgoals needed careful bullet management with `split_pures`.

**Resolution:** Used `+` bullets for each pure subgoal generated by `split_pures`, and applied `dump_pre_spatial` before each pure proof step to strip away spatial hypotheses.

## 4. `map_nth` default element mismatch
**Problem:** An initial attempt to use `unfold Znth; rewrite map_nth` failed because the default element in the goal was `0`, while `map_nth` introduces default `(fun x => x + 1) 0 = 1`.

**Resolution:** Instead of fighting with `nth_indep`, proved the dedicated `Znth_map_incr` lemma which handles the `Znth` default directly.
